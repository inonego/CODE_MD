# Story Craft

스토리 창작 지원 스킬. 4개 모듈(plot, character, world, scene)을 독립 또는 연계하여 사용한다.

## 모드 선택

인자에 `--mode`가 있으면 해당 모드로 진입한다. 모듈명만 있으면 자동으로 모듈 모드.
둘 다 없으면 `AskUserQuestion`으로 선택:

| 모드 | 언제 쓰는가 | 진행 방식 |
|------|------------|----------|
| **대화형** (interactive) | 아이디어가 막연할 때 | 씨앗 질문 → 확장 → 구체화 |
| **템플릿** (template) | 빠르게 골격을 잡을 때 | 개요 시트 채우기 |
| **모듈** (module) | 특정 영역만 작업할 때 | 모듈 선택 → 가이드 진행 |

---

## 대화형 모드

아이디어가 막연한 사용자와 대화하며 스토리를 점진적으로 구축한다.

### 진행 절차

0. **시작점 확인**: `AskUserQuestion`으로 기존 초안 여부를 확인한다
   - **처음부터 시작** → Step 1(씨앗 질문)부터 진행
   - **기존 초안이 있음** → 초안을 받아서(파일 경로 또는 텍스트) 분석 후:
     - 이미 확정된 요소를 정리하여 보여준다
     - 부족하거나 보강이 필요한 영역을 식별한다
     - Step 6(다음 단계 선택)으로 건너뛴다
1. **씨앗 질문**: "어떤 이야기를 하고 싶으세요? 떠오르는 이미지, 장면, 감정, 키워드 뭐든 좋습니다."
2. **파악**: 응답에서 장르, 분위기, 핵심 갈등, 주요 캐릭터 단서를 추출
3. **선택지 제공**: 파악된 정보가 부족하면 `AskUserQuestion`으로 핵심 요소를 좁힌다
4. **확장 질문**: 빠진 요소를 최대 4개까지의 질문으로 보충 (강요하지 않는다)
5. **구체화**: 충분한 정보가 모이면 관련 모듈 reference를 로드하여 구조화
6. **다음 단계 선택**: 구체화 결과를 보여준 뒤, 다음 작업 방향을 `AskUserQuestion`으로 제시
7. **출력**: 스토리 개요 시트 또는 해당 모듈의 출력 형식으로 정리

### 선택지 가이드

**Step 0 — 시작점 확인**:

- header="시작점", options=["처음부터 시작", "기존 초안이 있음"]
- **기존 초안이 있음** 선택 시: "초안을 붙여넣거나 파일 경로를 알려주세요."라고 안내

**Step 3 — 핵심 요소 좁히기** (씨앗 응답에서 파악 안 된 항목만 질문):

- **장르**: header="장르", options=["판타지", "SF/사이버펑크", "현대/일상", "사극/역사"] (Other로 자유 입력)
- **톤**: header="톤/분위기", options=["밝고 경쾌", "어둡고 무거움", "잔잔하고 서정적", "긴장감/스릴"]
- **갈등 유형**: header="핵심 갈등", options=["인물 vs 인물", "인물 vs 세계/사회", "인물 vs 자기 자신"]

> 이미 씨앗 응답에서 장르/톤이 분명하면 해당 질문을 건너뛴다.
> 한 번에 질문 1-2개만. 한꺼번에 몰아서 묻지 않는다.

**Step 6 — 다음 단계 선택**:

- header="다음 단계", options=["캐릭터를 더 파고 싶다", "플롯 구조를 잡고 싶다", "세계관을 설정하고 싶다", "씬/대사를 써보고 싶다"]
- 선택에 따라 해당 모듈 reference를 로드하고 모듈 모드로 자연스럽게 전환

### 핵심 원칙

- **흐름을 끊지 않는다**: 한 번에 질문 1-2개 이하. 사용자가 열정적이면 따라간다.
- **제안은 하되 강요하지 않는다**: "이런 방향은 어떨까요?" 형태로 제안
- **모듈은 자동 로드**: 캐릭터 이야기가 나오면 character.md, 배경이 나오면 world.md를 자연스럽게 참조

## 템플릿 모드

스토리 개요 시트를 제시하고, 사용자가 채운 항목 기반으로 빈 항목을 제안한다.

### 스토리 개요 시트

```markdown
## 스토리 개요

- **로그라인**: (한 줄 요약)
- **장르**:
- **톤/분위기**:
- **시대/배경**:
- **핵심 갈등**:
- **주인공**:
- **적대자/대립 세력**:
- **결말 방향**: (해피엔딩 / 비극 / 열린결말 / 미정)
```

### 진행 절차

1. 빈 시트를 제시한다
2. 사용자가 아는 항목을 채운다 (전부 채울 필요 없음)
3. 빈 항목에 대해 `AskUserQuestion`으로 선택지를 제공한다
4. 선택 결과 + 채워진 항목을 종합하여 나머지를 제안/자동완성한다
5. 사용자 확인 후, 다음 작업 방향을 `AskUserQuestion`으로 제시한다

### 선택지 가이드

**Step 3 — 빈 항목 채우기** (사용자가 비워둔 항목만 질문, 한 번에 최대 4개):

- **장르**: header="장르", options=["판타지", "SF/사이버펑크", "현대/일상", "사극/역사"]
- **톤/분위기**: header="톤/분위기", options=["밝고 경쾌", "어둡고 무거움", "잔잔하고 서정적", "긴장감/스릴"]
- **결말 방향**: header="결말 방향", options=["해피엔딩", "비극", "열린결말", "미정"]
- **갈등 유형**: header="핵심 갈등", options=["인물 vs 인물", "인물 vs 세계/사회", "인물 vs 자기 자신"]

> 사용자가 이미 채운 항목은 질문하지 않는다.
> 자유 입력이 더 적합한 항목(로그라인, 주인공 이름 등)은 텍스트로 질문한다.

**Step 5 — 다음 작업 방향**:

- header="다음 단계", options=["캐릭터 상세 설계", "플롯 구조 잡기", "세계관 설정", "씬/대사 작성"]
- 선택에 따라 해당 모듈 reference를 로드하고 이어간다

## 모듈 모드

원하는 모듈을 선택하여 해당 영역만 집중 작업한다.

### 모듈 목록

| 모듈 | 키워드 | Reference | 설명 |
|------|--------|-----------|------|
| **plot** | 플롯, 구조, 줄거리, 스토리라인 | [plot.md](references/plot.md) | 플롯/구조 설계 |
| **character** | 캐릭터, 인물, 등장인물, 프로필 | [character.md](references/character.md) | 캐릭터 설계, 나무위키식 프로필 |
| **world** | 세계관, 설정, 배경, 마법체계 | [world.md](references/world.md) | 세계관/설정 설계 |
| **scene** | 대사, 씬, 장면, 연출 | [scene.md](references/scene.md) | 대사/씬 작성 |

### 진행 절차

1. 모듈 선택: 인자에 없으면 `AskUserQuestion`으로 선택한다
2. 해당 reference 파일을 로드한다
3. 모듈 내 세부 작업을 `AskUserQuestion`으로 선택한다
4. reference의 가이드와 출력 형식에 따라 진행한다
5. 작업 완료 후 다음 단계를 `AskUserQuestion`으로 제시한다

### 선택지 가이드

**Step 1 — 모듈 선택** (인자 없을 때):

- header="모듈 선택", multiSelect=true
- options=["플롯/구조 설계", "캐릭터 설계", "세계관/설정", "대사/씬 작성"]

**Step 3 — 모듈 내 세부 작업** (모듈별로 다름):

- **plot**: header="플롯 작업", options=["구조 프레임워크 선택", "갈등 설계", "긴장 곡선/페이싱", "주제/로그라인 작성"]
- **character**: header="캐릭터 작업", options=["새 캐릭터 만들기", "기존 캐릭터 발전시키기", "관계도 설계", "나무위키식 프로필 출력"]
- **world**: header="세계관 작업", options=["세계 기반 설정", "규칙 체계(마법/기술) 설계", "역사/로어 작성", "세력/문화 설계"]
- **scene**: header="씬 작업", options=["새 씬 작성", "대사 작성/다듬기", "묘사/분위기 연출", "톤/무드 설정"]

**Step 5 — 작업 완료 후**:

- header="다음 단계", options=["같은 모듈에서 계속", "다른 모듈로 전환", "전체 정리/출력", "여기서 마무리"]

## 공통 규칙

### 일관성 체크

모듈을 넘나들며 작업할 때, 다음 일관성을 확인한다:

- **캐릭터 ↔ 플롯**: 캐릭터의 동기가 플롯의 행동과 일치하는가?
- **캐릭터 ↔ 세계관**: 캐릭터의 능력/배경이 세계 규칙 안에 있는가?
- **세계관 ↔ 씬**: 씬의 배경/분위기가 세계관 설정과 맞는가?
- **플롯 ↔ 씬**: 씬이 플롯 진행에 기여하는가?

불일치를 발견하면 사용자에게 알리고 조정 방향을 제안한다.

### 톤/무드 유지

- 작업 시작 시 장르와 톤을 확인한다
- 장르별 톤 가이드는 [scene.md](references/scene.md)의 "톤/무드 가이드" 참조
- 장르를 혼합할 때는 기본 톤 1개 + 보조 톤 1개로 제한 권장

### 모듈 간 교차 참조

작업 중 다른 영역이 필요한 시점을 감지하면 해당 reference를 추가 로드한다:

- 플롯 설계 중 캐릭터 동기가 필요하면 → character.md
- 캐릭터 설계 중 세계 규칙이 필요하면 → world.md
- 세계관 설계 중 구체적 장면이 필요하면 → scene.md
- 씬 작성 중 플롯 흐름 확인이 필요하면 → plot.md

### 스포일러 처리

모든 모듈에서 스포일러 요소를 다룰 때:

- `[스포일러]` 태그로 마킹한다
- 상세 규칙은 [character.md](references/character.md)의 "스포일러 처리 규칙" 참조
- 프로필, 관계, 대사, 행적에 각각 적용 방식이 다르다

### 영감 도우미

사용자가 창작에 막혔을 때 ("모르겠다", "아이디어가 없다", "뭐가 좋을까" 등) 자동으로 감지하여 영감을 제공한다.

**감지 시 동작**:

1. `AskUserQuestion`으로 영감 유형을 선택한다:
   - header="영감 유형", options=["랜덤 캐릭터 특성", "플롯 트위스트 아이디어", "배경/장소 아이디어", "갈등/긴장 아이디어"]
2. 선택된 유형에 맞는 아이디어 최대 4개를 제시한다
3. 사용자가 하나를 고르거나 조합하여 작업을 이어간다

> 아이디어는 현재까지 설정된 장르/톤/세계관과 일관성을 유지한다.
> 기존 설정이 없으면 범용적인 아이디어를 제시한다.
