# 보고서 출력 포맷

---

## 우려 시나리오 형식 (Phase 1 출력)

```
[도메인] 우려 #N: [제목]
- 상황: [어떤 맥락에서]
- 예상: [기대 동작]
- 우려: [실제로 일어날 수 있는 문제]
- 근거: [왜 이것이 우려인지 — 코드 구조, 아키텍처, 패턴 기반]
- 관련 코드: [파일/클래스/메서드 위치]
- 심각도: Critical / Warning / Info
```

---

## 검증 결과 형식 (Phase 2 출력)

```
[도메인] 이슈 #N: [제목]  (원래 우려 #N에서 확정)
- 상황: [...]
- 문제: [실제로 발생하는 문제]
- 코드 근거: [구체적 코드 경로와 조건]
- 영향: [이 문제가 게임에 미치는 영향]
- 심각도: Critical / Warning / Info
- 런타임 확인 필요: Yes / No
```

---

## 통합 보고서 형식 (최종 출력)

```markdown
## Unity QA 검사 결과

### 게임 프로필 요약
- 장르: [...]
- 핵심 루프: [...]
- 검사 모듈: [...]
- 검사 깊이: [understand / concern / verify / runtime]

### 요약
| 모듈 | Critical | Warning | Info |
|------|----------|---------|------|

### [모듈명] QA
#### 확정 이슈
| # | 제목 | 심각도 | 상황 | 문제 | 영향 |
|---|------|--------|------|------|------|

#### 기각된 우려
| # | 제목 | 기각 사유 |
|---|------|----------|

### 교차 참조 발견사항
(여러 모듈 결과를 교차 분석한 인사이트)

### 권장 조치
| 우선순위 | 이슈 | 조치 | 예상 영향 |
|---------|------|------|----------|
```

이슈가 없으면 게임 프로필 요약, 요약(모두 0), "이슈 없음"을 명시한다.
