# Visual QA — 사용자에게 올바른 시각적 경험을 제공하는가

UI가 의도대로 보이고 반응하며, 시각적 연출이 플레이를 방해하지 않는지를 판단한다.

---

## 우려 유형

### 1. UI 접근성 단절 (UI Inaccessibility)
- **관점:** 플레이어가 눌러야 할 버튼을 누를 수 없거나, 보여야 할 정보가 안 보이는 상황이 있는가?
- **살펴볼 곳:** UI 패널 열림/닫힘 조건, 버튼 활성화/비활성화 조건, EventSystem 존재 여부
- **판단 기준:** 특정 게임 상태에서 UI 인터랙션이 불가능해지는 경우. EventSystem이 없거나 중복되는 경우

### 2. UI 레이어 혼란 (Layer Conflict)
- **관점:** UI 요소가 겹쳐서 의도하지 않은 것이 앞에 오거나, 터치/클릭이 뒤의 요소로 전달되는가?
- **살펴볼 곳:** Canvas sortingOrder 할당 패턴, 동적 Canvas 생성, 팝업/오버레이 레이어 관리
- **판단 기준:** 동시에 활성화되는 Canvas 간 sortingOrder 관계가 불명확한 경우

### 3. 시각-로직 불일치 (Visual-Logic Mismatch)
- **관점:** 화면에 보이는 것과 실제 게임 상태가 다른 경우가 있는가?
- **살펴볼 곳:** HP 바가 실제 HP와 동기화되는지, 인벤토리 UI가 실제 데이터와 맞는지, 애니메이션 상태와 게임 상태
- **판단 기준:** UI 업데이트가 데이터 변경에 즉시 반응하지 않거나, 조건부로 누락되는 경우

### 4. 전환 연출 중 입력 문제 (Input During Transition)
- **관점:** 화면 전환(페이드, 로딩, 애니메이션) 중에 플레이어가 의도치 않은 입력을 할 수 있는가?
- **살펴볼 곳:** 씬 전환 중 버튼 클릭 가능 여부, 애니메이션 재생 중 입력 처리
- **판단 기준:** 전환 중 입력을 차단하는 로직이 없거나, 차단 해제 타이밍이 맞지 않는 경우

### 5. 해상도/비율 적응 (Resolution Adaptation)
- **관점:** 다양한 화면 크기에서 UI가 깨지거나 잘리는가?
- **살펴볼 곳:** Canvas Scaler 설정, 앵커 설정, Safe Area 처리
- **판단 기준:** Canvas Scaler 모드와 앵커 설정이 다양한 종횡비를 고려하지 않는 경우

### 6. 애니메이션 상태 교착 (Animation Deadlock)
- **관점:** 애니메이션이 특정 상태에서 멈춰 게임 진행이 불가능해질 수 있는가?
- **살펴볼 곳:** Animator 상태 머신의 전환 조건, Has Exit Time 설정, 애니메이션 이벤트 의존
- **판단 기준:** 전환 트리거가 발생하지 않을 수 있는 조건이 있거나, 애니메이션 이벤트가 없는 메서드를 호출하는 경우

---

## 코드 검증 관점

- UI 매니저/컨트롤러의 **Show/Hide 조건**을 추적하여 접근 불가 시나리오 확인
- Canvas sortingOrder 할당 패턴을 파악하여 **동시 활성 시 충돌** 가능성 확인
- 데이터 변경 → UI 업데이트 경로를 추적하여 **동기화 누락** 확인
- 씬 전환/애니메이션 중 **입력 차단 로직**의 범위와 타이밍 확인

---

## 런타임 검증 관점

- 주요 UI 화면에서 스크린샷 캡처 → 요소 겹침, 잘림, 깨짐 확인
- 활성 Canvas의 sortingOrder, RenderMode, EventSystem 상태 조회
- 게임 상태 변경 후 UI 표시값과 실제값 대조
- 전환 연출 중 버튼 클릭 시도 → 의도치 않은 동작 여부 확인
