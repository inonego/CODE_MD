## 최종 정리

Step 1 → 2 → 3 → 4 순서를 반드시 지키며, 각 Step을 **필수**로 실행한다. 건너뛰지 않는다.

### Step 1: 꾸밈 수준 결정

`AskUserQuestion`: 간결 / 표준 [기본] / 풍부

- **간결** — 헤더/리스트/볼드만
- **표준** — 표, 체크리스트, 구분선
- **풍부** — 표준 + 콜아웃, Mermaid, 뱃지

### Step 2: 실행 준비 확인

`[핵심]` 답변 완료 + 방향/구조/경계 명확성을 자동 확인한다. 불완전 시 해당 단계로 복귀.

### Step 3: 전체 계획 통합

앞서 만든 레이어 3개를 통합하여 **plan 파일에 작성**한다 (스코프별 "최종 계획서 구조" 참조).

- Q&A 의도/요구사항/제약을 "요구사항 및 설계 의도"로 별도 추출
- 도메인 가이드 로드 시 추가 검증 실행
- **채팅에는 완료 안내 + 한두 줄 요약만 출력** (다이어그램/그래프 출력 금지, 전체 내용은 plan 파일 참조)

### Step 4: 후속 조치

`AskUserQuestion` **필수**:

- **저장 및 실행** — 컨벤션 스킬 로드 (해당 시) → `ExitPlanMode`
- **저장만** — plan 파일 유지, `ExitPlanMode`
- **계획 재수정** — 해당 레이어로 복귀
- **저장하지 않고 종료** — plan 파일 삭제, `ExitPlanMode`
