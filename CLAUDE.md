# CLAUDE.md

## Workflow Rules

- 사용자가 '계획 먼저' 또는 계획 워크플로우를 요청하면, 질문을 **한 번에 하나씩** 대화형으로 진행한다. 질문을 한꺼번에 쏟아내거나, 계획 단계가 명시적으로 완료되기 전에 구현에 착수하지 않는다.
- 기존 설계 문서나 명세가 있으면 **먼저 읽고** 해결책을 제안한다. 사용자가 이미 문서화한 설계가 있는데 새로운 접근법을 만들어내지 않는다.

## Communication Rules

- 구조적/아키텍처적 변경 전에, 사용자의 의도를 **간결하게 재진술**하고 승인을 기다린다. 구조적 관계, 역할, 계층 구조를 추측하거나 가정하지 않는다.
- 응답은 간결하게, 핵심을 빠르게 전달한다. 사용자가 중간에 끊으면 너무 장황하거나 가치 전달이 늦다는 신호다. 긴 설명보다 **짧은 확인 + 행동**을 우선한다.

## Editing Rules

- 사용자가 특정 항목**만** 수정/변경/수정하라고 하면, 정확히 그것만 한다. 명시적으로 요청하지 않는 한 관련 파일이나 유사 항목으로 범위를 확장하지 않는다.

## Coding Rules

- 코드 변경 후 **컴파일 성공을 확인**한 뒤에야 수정 완료를 선언한다. 추측성 수정을 연쇄적으로 쌓지 말고, 각 변경마다 검증한다.
